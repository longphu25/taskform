# Deployment Guide

## Overview

TaskForm is deployed as a Walrus Site on Sui mainnet. The build pipeline produces
static assets with pre-compression (gzip + brotli), generates `ws-resources.json`
for optimal caching, and estimates deployment cost.

## Build Pipeline

```text
bun run build:walrus
  ├── tsc -b                          (type check)
  ├── VITE_BASE_PATH=/ vite build     (bundle with compression)
  │   ├── dist/*.html, dist/assets/*  (original files)
  │   ├── dist/**/*.gz                (gzip pre-compressed)
  │   └── dist/**/*.br                (brotli pre-compressed)
  └── gen-ws-resources.ts             (walrus site config + cost estimate)
      └── dist/ws-resources.json
```

### Commands

| Command | Purpose |
|---------|---------|
| `make build` | Standard build (GitHub Pages, base=/taskform/) |
| `make build-walrus` | Walrus Site build (base=/, generates ws-resources.json) |
| `make deploy-walrus` | Build + start Docker deploy container |

## ws-resources.json

Auto-generated by `scripts/gen-ws-resources.ts`. Contains:

- **Headers**: `Cache-Control: max-age=31536000, immutable` for all hashed assets
  (JS/CSS/WASM in `/assets/`), `no-cache` for HTML entry points
- **Routes**: Multi-page SPA fallback per page
- **Ignore**: `*.gz`, `*.br` files excluded from Walrus upload
- **Metadata**: Site name, description, project URL
- **Cost estimate**: Printed to console after generation

### Customizing epochs

```bash
bun run scripts/gen-ws-resources.ts 200  # 200 epochs ≈ 93 months
```

Default is 100 epochs (~47 months).

## Walrus Site Deploy (Docker)

Deploy uses a disposable Docker container to avoid installing `site-builder`
locally and to keep private keys ephemeral.

### Prerequisites

- Docker (OrbStack or Docker Desktop)
- SUI + WAL tokens on mainnet wallet

### Flow

```bash
# 1. Build + start container
make deploy-walrus

# 2. SSH into container
ssh root@localhost -p 2222
# Password: deploy

# 3. Inside container: import wallet
sui client new-env --alias mainnet --rpc https://fullnode.mainnet.sui.io:443
sui client switch --env mainnet
sui keytool import <PRIVATE_KEY_BASE64> ed25519
sui client switch --address <YOUR_ADDRESS>

# 4. Verify balance
sui client active-address
sui client balance

# 5. Deploy (first time creates new site)
site-builder deploy --epochs 100 /site

# 6. Note the object ID from output, then exit
exit

# 7. Stop + auto-remove container (private key gone)
docker stop walrus-site-deploy
```

### Subsequent deploys

After first deploy, `ws-resources.json` stores the `object_id`. The script
preserves it across rebuilds, so `site-builder deploy` will update the existing
site automatically.

### Security

- Container runs with `--rm` flag (auto-deleted on stop)
- `dist/` mounted read-only (`:ro`)
- SSH password is trivial (`deploy`) — only exposed on localhost:2222
- No persistent volumes — private key exists only in container memory
- After `docker stop`, all traces of the key are gone

## Cost Estimation

The build script calculates estimated costs based on:

- **Storage**: $0.023/GB/month (Walrus fixed USD pricing, paid in WAL)
- **SUI gas**: ~0.15 SUI for 3-5 transactions (site-builder uses Quilt batching)
- **Epoch duration**: ~14 days on mainnet

Example output for current build (~2 MB, 100 epochs):

```
💰 Cost Estimate (100 epochs ≈ 47 months):
  Files to upload: 47 (2.0 MB)
  Storage cost:    ~$0.0021 USD (paid in WAL)
  SUI gas:         ~0.15 SUI (3-5 transactions)
```

For exact WAL amount, check: https://costcalculator.wal.app/

## Compression Strategy

| Target | Strategy |
|--------|----------|
| GitHub Pages | Serve `.gz`/`.br` via CDN (auto content-negotiation) |
| Walrus Site | Upload originals only, skip `.gz`/`.br` (no negotiation) |

Walrus Site portal serves files as-is without content-negotiation. The
`ws-resources.json` ignore patterns ensure compressed duplicates are not
uploaded, saving WAL storage cost.

## Network Configuration

| Component | Network | Notes |
|-----------|---------|-------|
| Move contract | Testnet | Package ID in AGENTS.md |
| Walrus Site hosting | Mainnet | Requires SUI + WAL on mainnet |
| Walrus blob storage | Mainnet | Form schemas + submissions |
| Frontend RPC | Testnet | SuiGrpcClient points to testnet |

## Docker Image Contents

The deploy image (`deploy/Dockerfile`) includes:

- `site-builder` (mainnet-latest) — Walrus Site publisher
- `walrus` (mainnet-latest) — Walrus CLI
- `sui` (latest) — Sui CLI for wallet management
- `openssh-server` — SSH access for key import
- `sites-config.yaml` — Pre-configured for mainnet

## Accessing the Deployed Site

After deploy, convert the object ID to a Base36 subdomain:

```bash
site-builder convert <OBJECT_ID>
# Output: <base36-string>
```

Site is live at: `https://<base36-string>.wal.app`
